# 交互规范

## 0. 文档控制

- 版本：v0.2
- 状态：Draft
- 用途：定义本文件的职责、范围与使用方式
- 适用对象：产品、设计、研发、测试、运维
- 单一来源：本文件
- 规范等级：MUST / SHOULD / MAY



## 1. 交互原则

- 用户主要在外部 Hermes 中对话，ScriptHub 主要负责在统一工具中心里管理插件 / 脚本工具和 Hermes 沉淀工具，并记录参数、运行、产物、确认和修复
- Hermes 先给计划，再调用 ScriptHub 工具执行
- Hermes 先暴露风险，再请求用户确认
- ScriptHub 先显示用户能理解的工具类型、参数、运行历史、产物和 Hermes 修复，再把 ToolCall / Trace / Audit 放入高级层
- ScriptHub 先说明读写范围、覆盖风险和确认状态，再允许外部工具调用不可逆动作
- 所有长任务都必须可中断、可恢复、可回放

## 2. 全局交互规则

### 2.1 异步任务

- 所有由 Hermes 发起的执行类工具调用必须显示进度
- 长任务必须有运行中、等待中、失败中、完成状态
- 失败必须给出可操作建议
- 页面不应要求用户通过复杂表单推进首次常规执行流程；首次执行主要由 Hermes 对话完成
- 已沉淀工具、脚本、插件和外部服务工具都可以由用户在工具中心打开对应工具的桌面级置顶小窗，调整参数后再次运行

### 2.2 审批

- 高风险动作必须进入审批态
- 审批前必须可见风险说明、读取范围、写入范围、覆盖范围和影响对象
- 审批后必须记录审计事件
- 审批优先在 Hermes 对话中完成；ScriptHub 只显示审批请求、来源、结果和审计记录

### 2.3 错误处理

- 错误必须区分：连接错误、执行错误、权限错误、超时错误、结果不确定
- 错误页面必须提供下一步动作，例如让 Hermes 修复、重试、修改路径、重新连接、重新选择对象、跳过、人工处理
- Hermes 修复成功后，修复过程必须进入该工具的运行历史或改进记录

### 2.4 空态

- 空态必须告诉用户“先在 Hermes 里完成一次操作，跑通后这里会生成你的工具”
- 空态不应默认提供手动生产操作按钮，调试入口必须标注为开发 / fallback

## 3. 页面级规则

### 3.1 首页

- 个人默认首页显示工具中心、工具类型、最近运行、可调参数、最近产物、待确认和 Hermes 修复
- 工具小窗不是常驻全局面板；它由用户在工具中心点击某个工具后打开，承载该工具的参数、确认、执行、产物和 Hermes 修复。正式产品目标是脱离浏览器的桌面置顶窗口；Web 内窗口只作为 MVP fallback
- 不默认暴露 ToolCall、Trace、Audit 原始信息；这些进入高级视图
- 不提供正式“新建任务”主入口；结构化任务入口只作为调试 fallback

### 3.2 任务详情

- 左侧看任务目标与步骤
- 中间看执行时间线与事件
- 右侧看摘要、风险与相关资产

### 3.3 工作流

- 画布是主要操作区
- 节点属性、失败策略、权限规则放在右侧
- 运行日志和校验结果固定在底部

### 3.4 Trace / Review / Approval

- Review 优先看待审内容
- Approval 优先看风险说明
- Trace 优先看执行链
- Replay 优先看关键点跳转
- 所有审批动作必须能回溯到 Hermes 对话、ToolCall 和 Trace

### 3.5 资产详情

- 首屏优先看预览
- 其次看来源链
- 再看版本差异

## 6. 关键页面交互细则

### 6.1 首页

- 顶部用于显示外部 Hermes 会话、连接状态和当前工具运行状态
- 中部优先展示工具中心、能力分类、成熟度、来源标签、工具参数、运行历史和最近产物
- 右侧用于待确认、Hermes 修复、连接器健康和工具候选
- ToolCall 时间线、Trace 和 Audit 只在高级模式或详情页中展示

### 6.2 任务中心

- 支持筛选、排序、搜索
- 点击任务先进入详情页，不直接跳出执行
- 批量操作必须显示影响范围

### 6.3 任务详情

- 任务目标、计划、执行流、结果应分区清晰
- 所有失败节点必须能定位到对应事件
- 所有审批节点必须能返回到审批记录

### 6.4 工作流页

- 节点拖拽后必须立即校验
- 连线错误必须显式提示
- 节点属性修改后必须有保存反馈

### 6.5 Trace / Review / Approval

- Review 只负责看待审内容
- Approval 负责解释确认请求，包括读取、写入、覆盖、风险和回滚可能性；正式确认优先回到 Hermes 对话完成
- Trace 负责解释执行链
- Replay 负责定位过程节点

## 7. 页面状态表建议

每个页面都至少定义以下状态：

- default
- loading
- empty
- partial
- running
- waiting approval
- disconnected
- error
- read only

## 8. 交互禁区

- 不允许执行过程没有反馈
- 不允许高风险动作没有审批
- 不允许失败后不给下一步
- 不允许关键数据只靠悬浮提示
- 不允许把 ScriptHub 做成绕过 Hermes 的主操作后台
- 不允许个人默认界面要求用户理解 ToolCall、descriptor、trace_id、audit_id 才能使用产品
- 不允许工具在未展示读写范围和覆盖风险时直接执行正式生产动作
- 不允许把 Hermes 修复过程丢掉；修复成功后必须能回到工具历史或工具版本
- 不允许把插件 / 脚本工具做成无记录的“裸执行按钮”；它也必须进入参数、确认、运行历史和产物记录
- 不允许把工具小窗做成常驻控制台；它必须由具体工具触发，并围绕当前工具上下文展示
