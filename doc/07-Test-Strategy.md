# 测试策略

## 0. 文档控制

- 版本：v0.2
- 状态：Draft
- 用途：定义本文件的职责、范围与使用方式
- 适用对象：产品、设计、研发、测试、运维
- 单一来源：本文件
- 规范等级：MUST / SHOULD / MAY



## 1. 目标

确保系统可长期维护、可回归、可验证。

## 2. 测试层级

- 单元测试
- 契约测试
- 集成测试
- UI 测试
- 回放测试
- 沙箱测试
- 红线测试

## 3. 核心测试对象

- Runtime 状态机
- API Contract
- Skill / Tool schema
- Connector 生命周期
- Workflow 运行
- Event 生成
- Asset 追溯

## 4. 必测场景

- 计划生成
- 审批通过
- 审批拒绝
- Connector 断线
- 任务重试
- 任务回滚
- 结果不确定
- 回放一致性

## 5. 质量门槛

- 关键契约必须通过
- 红线用例必须通过
- 回放结果必须可对比
- 失败必须可归因

## 6. 结果输出

- 测试摘要
- 失败案例
- Diff 报告
- 复现步骤
- 回归结论

## 8. 退出标准

- 关键契约测试全部通过
- 核心场景回放一致
- 红线测试无阻断问题
- 高风险操作通过人工检查
