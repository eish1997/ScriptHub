# 仓库实施说明

## 0. 文档控制

- 版本：v0.1
- 状态：Draft
- 用途：告诉 AI agent 和研发团队如何在仓库中落地实现
- 适用对象：研发、AI agent、测试、产品
- 单一来源：本文件
- 规范级别：MUST / SHOULD / MAY

## 1. 目标

本说明用于解决“文档已经写好，但不知道从仓库哪里开始改”的问题。

## 2. 实施原则

- 先看目录，再看约束
- 先看现有代码，再改结构
- 先补骨架，再补细节
- 先做最小闭环，再做扩展能力

## 3. 仓库阅读顺序

### 3.1 首先看

- 根目录结构
- 启动方式
- 构建方式
- 测试方式
- 现有组件库

### 3.2 其次看

- 页面路由
- 全局状态管理
- API 封装
- 设计系统
- 现有文档映射

### 3.3 最后看

- 具体业务页面
- 复杂工作流
- Connector 接入
- 审批与审计逻辑

## 4. 目录约定建议

- `src/pages`：页面
- `src/components`：通用组件
- `src/modules`：业务模块
- `src/services`：API 和数据访问
- `src/models`：数据模型与状态
- `src/styles`：全局样式和 tokens
- `src/tests`：测试

## 5. AI agent 修改规则

- 先确认要改的文件范围
- 不要同时改多个重叠模块
- 不要改和任务无关的代码
- 每次修改后要说明变更文件
- 每次修改后要说明如何验证

## 6. 推荐开发顺序

### 6.1 第一轮

- 路由
- 布局
- 组件骨架
- 空态

### 6.2 第二轮

- 任务中心
- 任务详情
- Trace 面板
- Connector 面板

### 6.3 第三轮

- 事件与审计
- 资产详情
- 评估与验证
- 设置与权限

## 7. 验收方式

- 能跑起来
- 能打开页面
- 能看到正确状态
- 能完成一个最小任务闭环
- 能通过基础测试
