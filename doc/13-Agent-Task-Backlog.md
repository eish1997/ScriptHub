# Agent 任务清单

## 0. 文档控制

- 版本：v0.1
- 状态：Draft
- 用途：把 MVP 拆成 AI agent 可以直接执行的具体任务
- 适用对象：AI agent、研发、测试、产品
- 单一来源：本文件
- 规范级别：MUST / SHOULD / MAY

## 1. 使用原则

- 每个任务必须足够小
- 每个任务必须有明确输入和输出
- 每个任务必须能独立验收
- 每个任务必须有依赖说明
- 每个任务必须避免和其他任务重叠修改同一文件

## 2. 任务模板

每个任务应包含：

- 任务编号
- 任务标题
- 背景
- 目标
- 输入
- 输出
- 依赖
- 修改范围
- 验收标准
- 风险

## 3. 推荐任务拆分

### 3.1 前端骨架任务

- 搭建首页布局
- 搭建任务中心列表
- 搭建任务详情页结构
- 搭建 Trace / Review / Approval 面板
- 搭建资产详情页

### 3.2 状态与数据任务

- 接入任务状态模型
- 接入页面状态矩阵
- 接入空态、加载态、错误态
- 接入连接器健康状态

### 3.3 契约任务

- 接入 API 响应结构
- 接入错误模型
- 接入 trace_id 透传
- 接入审计记录展示

### 3.4 观测任务

- 接入事件列表
- 接入 trace 时间线
- 接入审计详情
- 接入资产来源链

### 3.5 基础能力任务

- Skill 列表页
- Tool 列表页
- Connector 面板
- 版本与发布信息

## 4. 任务优先级

### P0

- 首页
- 任务详情
- Trace / Review / Approval
- 连接器状态

### P1

- 资产详情
- 事件与审计
- Skill / Tool 列表

### P2

- 设置与权限
- 版本与发布
- 评估与验证

## 5. Agent 执行规则

- 一次只做一个任务
- 做完必须写验收结果
- 不允许擅自扩大范围
- 不允许修改无关文件
- 不允许跳过测试

## 6. 推荐任务示例

### A-001 首页布局

- 输入：UI 主页规范
- 输出：首页路由、布局、空态
- 验收：页面可打开，信息分区正确

### A-002 任务详情页

- 输入：任务详情页规范
- 输出：任务目标、时间线、右侧摘要
- 验收：能展示任务状态和执行信息

### A-003 Trace 面板

- 输入：Trace / Review / Approval 规范
- 输出：Review 队列、Approval Drawer、Trace 时间线
- 验收：能查看并切换审查信息
