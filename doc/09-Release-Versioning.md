# 版本与发布策略

## 0. 文档控制

- 版本：v0.2
- 状态：Draft
- 用途：定义本文件的职责、范围与使用方式
- 适用对象：产品、设计、研发、测试、运维
- 单一来源：本文件
- 规范等级：MUST / SHOULD / MAY



## 1. 目标

让 Runtime、UI、Contract、Skill、Tool、Connector 可以长期演进而不互相打穿。

## 2. 版本原则

- 先兼容，再升级
- 先弃用，再删除
- 先灰度，再全面切换

## 3. 版本对象

- Runtime
- UI
- Skill
- Tool
- Connector
- Workflow
- Event schema

## 4. 需要定义的规则

- 主版本 / 次版本 / 修订版本
- 向后兼容范围
- 弃用周期
- 回滚策略
- 默认版本选择

## 5. 发布流程

- 开发完成
- 契约校验
- 测试通过
- 灰度发布
- 全量发布
- 观测与回滚预案

## 6. 推荐补充

- 迁移脚本规则
- 特性开关规则
- 旧版本回放兼容规则

## 7. 发布层级

- Draft
- Internal
- Beta
- Stable
- Deprecated

## 8. 升级策略

- 先升级契约，再升级实现
- 先小范围灰度，再全量发布
- 先保留旧接口，再逐步淘汰

## 9. 回滚策略

- 版本必须可快速回滚
- 回滚必须保留审计记录
- 回滚后必须重新验证关键场景

## 10. 建议补充

- 迁移说明
- 发布清单
- 发布检查表
- 兼容性公告模板
