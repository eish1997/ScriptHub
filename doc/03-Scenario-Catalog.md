## 1. 场景用途

场景库用于统一沉淀可复用的典型生产路径，帮助产品、设计、开发、测试对齐。第一版场景必须从个人用户可感知价值出发：ScriptHub 是跨 DCC 工具中心，既承载插件 / 脚本工具，也承载 Hermes 完成复杂操作后沉淀出的可调参数、可再次执行、可持续完善的工具。

## 2. 基础场景

### 2.1 导出 FBX

- 入口：用户在外部 Hermes 中提出“把当前 Maya 选择导出 FBX”
- Hermes：拆解意图、检查 Connector、请求必要确认，并在出错时持续追问、重试和修复
- ScriptHub：接收 Hermes ToolCall，创建任务、写入 trace，并在个人首页形成一条操作历史
- 执行：Connector 导出文件
- 结果：生成 FBX，记录来源链，并生成“导出当前选择为 FBX”的工具候选

### 2.2 插件 / 脚本工具进入工具中心

- 入口：Hermes 已写好一个 Maya Python 导出脚本，或用户上传一个 Blender 批处理脚本
- ScriptHub：解析工具 manifest、参数、适用 DCC、权限、风险和版本
- 用户看到：工具中心新增一个工具，来源标签为脚本或 DCC 插件，成熟度显示为已验证或团队推荐
- 使用方式：用户在 ScriptHub 工具中心点击该工具，打开桌面级置顶工具小窗，在窗口中修改参数并执行，而不是必须打开 DCC 内置插件面板
- 执行：ScriptHub 通过 Connector 调用 Maya / Blender / Unreal 执行底层脚本
- 结果：运行历史、产物、错误和 Hermes 修复记录仍进入同一套工具体系

### 2.3 首次跑通并生成 Hermes 工具

- 入口：用户第一次在 Hermes 中要求“把这个 Maya 文件导出成 FBX”
- 过程：可能出现路径不可写、选择为空、Connector 断开、导出参数缺失、覆盖风险等问题
- Hermes：持续介入修复，必要时追问用户、修改参数、重连 Connector 或换输出路径
- ScriptHub：记录每次失败、修复动作、确认点、最终成功参数和产物
- 结果：用户得到一个“导出 FBX”工具，而不是只得到一次导出结果

### 2.4 工具参数化复用

- 入口：Hermes 已成功完成一次“导出当前选择为 FBX”
- Hermes：返回执行结果和关键参数
- ScriptHub：抽取输入对象、输出路径、覆盖策略、使用工具、确认点、失败处理和产物记录
- 用户看到：我的工具新增“导出当前选择为 FBX”
- 下次使用：用户点击按钮或在 Hermes 中说“再跑一下上次那个导出”
- 参数：用户可修改输出路径、是否覆盖、导出范围、命名规则等
- 结果：不需要重新长篇描述，执行前仍显示读写范围和风险

### 2.5 工具再次运行前安全确认

- 入口：用户准备再次运行已沉淀工具
- ScriptHub：展示会读取的选择对象、会写入的文件路径、是否覆盖现有文件、风险等级和所需权限
- Hermes：在对话中同步解释确认内容
- 用户：确认、拒绝或修改输出路径
- 结果：确认后执行；拒绝后不产生副作用；修改路径后生成新的运行记录

### 2.6 再次运行失败与 Hermes 修复

- 入口：工具再次运行失败，例如输出路径不可写或 Maya 未选择对象
- ScriptHub：把错误转为用户可执行的修复动作
- 修复动作：重试、修改路径、重新连接 Connector、重新选择对象、跳过、人工处理
- Hermes：可以继续对话式指导用户修复
- 结果：修复动作、Hermes 解释、再次运行结果进入同一个工具的运行历史
- 增强：如果 Hermes 找到通用修复方式，应更新工具参数、失败处理或版本

### 2.7 Hermes 工具升级为插件 / 脚本

- 入口：某个 Hermes 工具已经高频、稳定、边界清楚
- Hermes：将工具步骤固化为 Maya / Blender / UE 脚本或插件，并生成 manifest
- ScriptHub：把它注册为插件 / 脚本工具，同时保留原 Hermes 工具的运行历史和来源
- 用户：仍在该工具小窗里调参数并执行
- 结果：工具获得更稳定的底层实现，但不丢失运行历史、安全确认和 Hermes 修复能力

### 2.8 继续完善工具

- 入口：用户对 Hermes 说“这个导出工具增加一个贴图一起拷贝的选项”
- Hermes：修改工具参数模板、补充执行步骤、验证新路径；如果是插件 / 脚本工具，则可以修改代码并生成新版本
- ScriptHub：记录工具版本、参数变化、验证结果和回滚点
- 结果：同一个工具变强，而不是新增一堆一次性脚本

### 2.9 产物来源追踪

- 入口：用户查看某个 FBX 文件
- ScriptHub：显示它来自哪个 Hermes 会话、哪个工具、哪次运行、什么输入对象、哪个输出路径和哪个版本
- 用户：可以打开文件位置、查看上次运行、再次运行或复制参数
- 结果：个人用户能知道文件从哪里来；团队用户可进一步查看 Trace / Audit

### 2.10 Maya 到 UE

- 入口：用户在 Hermes 中描述流转目标
- Hermes：调用 ScriptHub 查询资产、生成计划、触发导出和导入工具
- ScriptHub：记录跨工具 trace、审批点和资产版本
- 结果：UE 可用资产、个人可见工具、团队可见审计记录和 workflow 候选

### 2.11 批量处理

- 入口：用户在 Hermes 中描述批量规则
- Hermes：将规则转为多次 ToolCall 或批处理 ToolCall
- ScriptHub：展示批处理历史、产物、失败项和恢复建议
- 结果：批处理报告和技能候选参数模板

### 2.12 高风险操作审批

- 入口：Hermes 准备调用高风险工具
- Hermes：在对话中向用户说明影响范围并请求确认
- ScriptHub：生成确认对象，展示读取范围、写入范围、覆盖风险、ToolCall 和等待状态
- 执行：Hermes 收到确认后调用审批 / 继续执行工具
- 结果：审批记录 + 执行记录 + 对话来源

### 2.13 断线恢复

- 输入：Connector 断开事件
- Hermes：收到状态变化后决定重试、追问或暂停
- ScriptHub：显示断线状态、最近工具运行、修复动作和可恢复 trace
- 执行：Hermes 调用健康检查或重连工具
- 结果：恢复成功或明确失败原因
